# 知识图谱扩展方向探讨

本文档探讨了当前知识图谱的四个潜在扩展方向，旨在为未来的开发提供指导。

## 当前知识图谱结构概述

核心是 `Stock` 节点，它包含了股票的基本信息和一些财务快照。`Industry` (行业), `Area` (地域), `Market` (板块), `Exchange` (交易所) 作为 `Stock` 的基本分类维度。`Concept` (概念题材) 是一个关键的扩展点，一只股票可以属于多个概念，一个概念也包含多支股票。已有一套从 `data_engineering` 模块同步数据到知识图谱的机制，支持全量和增量。

## 潜在扩展方向

### 方向 A：丰富 `Stock` 节点的内在属性

此方向侧重于深化单个 `Stock` 节点的信息量。

*   **具体内容**：
    *   加入更多的财务指标，例如：现金流、成长性指标（如营收增长率、利润增长率）、运营效率指标等。
    *   增加公司层面的结构化信息，例如：股东信息（持股比例、股东类型）、高管信息（姓名、职位、任职时间）、股权质押情况等。
    *   补充非财务信息，例如：公司治理评级、ESG（环境、社会、治理）评级。
*   **预期价值**：增强对单个公司的理解深度，支持更精细的个股筛选和基本面分析。
*   **挑战**：需要稳定可靠的数据源支持，并可能增加数据同步和存储的复杂度。

### 方向 B：增加新的节点类型和关系

此方向旨在扩展图谱的广度，引入与股票关联的全新实体类型，从而揭示新的洞察。

*   **具体内容**：
    *   引入 `Person` 节点：代表公司高管、创始人、核心技术人员等，并建立如 `MANAGES` (管理), `FOUNDED` (创立), `HOLD_POSITION` (任职) 等关系。
    *   引入 `Product` 或 `Service` 节点：代表公司的主要产品或服务，并建立 `PRODUCES` (生产), `OFFERS` (提供) 等关系。
    *   引入 `Event` 节点：代表对公司有重大影响的事件，例如重大合同签订、并购、政策变化等，并建立 `AFFECTS` (影响), `CAUSES` (导致) 等关系。
*   **预期价值**：能够回答“哪些股票由同一个传奇 CEO 管理过？”或“某款爆品关联了哪些上市公司？”这类非常有洞察力的问题，实现图谱结构的质的飞跃。
*   **挑战**：需要发现并集成多种类型的数据源，数据抽取和实体链接（Entity Linking）的复杂度较高。

### 方向 C：扩展 `Concept` 的维度（构建概念产业链）

此方向专注于深化 `Concept` 节点间的关系，将独立的“概念标签”升级为“概念网络”。

*   **具体内容**：
    *   建立 `Concept` 之间的关系：例如 `IS_UPSTREAM_OF` (上游), `IS_DOWNSTREAM_OF` (下游), `COMPETES_WITH` (竞争), `IS_PART_OF` (组成部分), `ENABLER_FOR` (技术驱动) 等。例如，`"新能源车"` **上游是** `"锂电池"`。
    *   为 `Concept` 节点增加动态属性：例如热度指数、市场情绪、近期新闻提及度等，以便进行趋势分析。
*   **预期价值**：能够从“点的集合”升级到“网的结构”，支持产业链分析、主题轮动分析、宏观风险传导等更高级的玩法。这是当前图谱中最具战略价值的演进路径。
*   **挑战**：获取高质量的“概念间关系”数据是核心难点，可能需要结合专家知识、LLM辅助或高级NLP关系抽取技术。

### 方向 D：引入外部数据源（如宏观经济、供应链）

此方向旨在将图谱与更广泛的外部世界连接，提供更全面的分析视角。

*   **具体内容**：
    *   整合宏观经济数据：引入 `MacroEconomyIndicator` 节点（如 `GDP`, `CPI`, `PMI`），并建立 `INFLUENCES` (影响) 等关系。
    *   整合供应链数据：建立 `Stock` 节点之间的 `IS_SUPPLIER_OF` (是...的供应商) 或 `IS_CUSTOMER_OF` (是...的客户) 关系，构建企业间的供需网络。
    *   整合政策数据：引入 `Policy` 节点，并建立 `SUPPORTS` (支持), `RESTRICTS` (限制) 等关系。
*   **预期价值**：能够进行更全面的情景分析，例如宏观经济政策对产业链的传导效应、供应链中断对企业的影响、替代性风险等。
*   **挑战**：数据源通常分散且获取成本高昂，数据清洗、标准化和实体链接的复杂度非常大。特别是供应链数据，往往涉及商业敏感信息，获取难度极大。
