# 灵感：概念题材图谱 (Concept Graph Idea)

**核心哲学**：概念题材是“流动”的、 “鲜活”的，甚至是“带有情绪”的。这不同于僵硬的行业分类。
**目标**：捕捉市场脉搏，支持“投机为矛”的策略。

---

## 1. 数据深度：不仅仅是“标签” (Beyond Tags)

如果只记录 `(Stock)-[:BELONGS_TO]->(Concept)` 是不够的，必须引入**权重**与**元数据**。

*   **关联强度 (Weight)**：
    *   **龙头 vs 跟风**：万丰奥威是低空经济的**龙头**（权重 1.0），而某某公司只是**蹭热点**（权重 0.2）。
    *   *Implementation Idea*: 在关系中增加权重属性 `(Stock)-[:BELONGS_TO {weight: 0.9}]->(Concept)`。
    *   *Value*: Research 模块在分析时，能区分核心标的和边缘标的，甚至基于权重构建不同的投资组合策略。

*   **入选理由 (Reason)**：
    *   *Context*: 为什么它属于这个概念？是因为“子公司生产飞行汽车部件”还是“与某某签署了战略协议”？
    *   *Implementation Idea*: 将理由作为属性存入关系中，或者作为 Text 供 LLM 阅读。

## 2. 时间维度：概念生命周期 (Lifecycle)

概念题材具有极强的时效性，今天炒作明天即凉。

*   **概念热度 (Heat Index)**：
    *   每个 Concept 节点应该有一个动态的 `heat_score`。
    *   *Calculation*: 基于该概念下所有成份股的**成交量、换手率、涨幅**来实时计算。
    *   *Trigger*: 当系统发现某个 Concept 的热度突然飙升（Slope > 阈值），自动触发 **Catalyst Researcher** 生成突发研报。

*   **纳入时间 (Timestamp)**：
    *   记录 `created_at`。
    *   *Strategy*: 刚纳入概念的新面孔往往爆发力最强（比如某公司突然宣布切入人形机器人赛道）。

## 3. 结构层级：概念树 (Concept Tree)

概念往往是嵌套的，构建层级关系有助于宏观判断。

*   **层级关系**：
    *   `固态电池` -> `锂电池` -> `新能源车`。
    *   *Implementation Idea*: 引入 `(Concept A)-[:SUB_CONCEPT_OF]->(Concept B)` 关系。
    *   *User Scenario*: 当用户问“新能源车还有机会吗？”时，图谱可以向下钻取，告诉他“整车不行了，但固态电池这个细分还在爆发”。

## 4. 数据源策略 (Data Source Strategy)

*   **Level 1 (基础版)**: Tushare/AKShare 的概念分类接口。简单、标准，但可能滞后。
*   **Level 2 (进阶版)**: 爬取同花顺/东财的“板块资金流向”数据。
*   **Level 3 (AI 版)**: 让 LLM 扫描财经新闻标题，自动提取新概念。
    *   *Input*: "某公司宣布研发出人形机器人伺服电机"
    *   *Output*: 自动将该公司关联到 `人形机器人` 概念。

---

## 5. 场景构想 (Scenario: The Opportunity Hunter)

**工作流示例**：
1.  **监控**：系统扫描发现 `固态电池` 概念节点的热度指数从 30 飙升到 80。
2.  **触发**：Coordinator 唤醒 **Catalyst Researcher (催化剂专家)**。
3.  **图谱检索**：专家查询图谱，找到 `固态电池` 下权重 > 0.8 的核心票，且近期没有大涨的（基于 Stock 节点的行情数据）。
4.  **推理**：发现某只票 A 是另一只龙头票 B 的供应商（复用供应链关系）。
5.  **输出**：生成机会提示——“板块爆发 + 补涨逻辑 + 供应链传导”。
