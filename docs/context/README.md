# Context & Inspiration (灵感与上下文)

本目录用于存放系统的各类上下文信息、演进路线图以及尚未正式立项的灵感碎片。

## 目录结构 (Directory Structure)

### 1. Roadmap (演进规划)
`docs/context/roadmap/`
存放系统级、模块级的长期演进路线图。
- 示例: `knowledge_center.md` (知识中心演进路线)

### 2. Features (功能探索)
`docs/context/features/`
针对具体功能模块的早期探索、RFP (Request for Proposal) 或草案。
- 示例: `market_insight_exploration.md` (市场洞察模块探索)

### 3. Topics (主题研究)
`docs/context/topics/`
跨模块的通用主题研究、领域模型探讨或纯粹的灵感记录。
- 示例: `concept_graph.md` (概念图谱模型研究)

### 4. Inbox (灵感收集箱)
`docs/context/inbox/` (Optional)
临时存放未经整理的灵感、聊天记录片段等。

---

> **Tip**: 当某个 Feature探索成熟并决定实施时，应将其转化为正式的 OpenSpec (`openspec/changes/...`) 或移动到 `docs/design/` 下作为正式设计文档。
